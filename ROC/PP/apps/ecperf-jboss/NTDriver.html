Driving an ECPerf Workload from an NT environment

To drive ECPerf on a System Under Test (SUT) a batch file is provided that will allow a Windows NT system as the driver. The batch file is called Driver.bat. Several environment variables that pertain the to both the driving system and the SUT.

To use this batch file edit it and modify the various entries for environment variables. These variables define the location of various files, classes and driving configuration. For each of the variables a brief explanation of the variable is provided along with a recommended setting if appropriate.

Once the updates are completed the batch file can be invoked to start a run. There are four stages to a run. Each stage is separted from the other with a simple Pause in the batch file. Once the stage is complete press enter and the next stage will begin.

1. First the RMI registry is started.

2. Start the controller.

3. Start the various domain daemons. They include the Orders, Manufacturing and Large Order Line.

4. Finally the driver itself is started.

For more general run instructions please refer to the driver instructions provided with the kit.
