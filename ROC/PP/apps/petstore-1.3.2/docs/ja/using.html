デモの概要

Java Pet Store Demo は、次の主要なコンポーネントで構成されています。

  • Storefront: Storefront は、Web フロントエンドでの主要なユーザインタフェースです。顧客は、Storefront を使用してペットを注文します。

  • OPC (Order Processing Center): OPC は、Storefront からの注文を受け取ります。

  • Supplier: Supplier は、OPC からの注文を在庫から充当し、OPC に納品書を送付します。

  • Admin: Admin は、JFC/Swing フロントエンドの管理者インタフェースです。管理者は、Admin を使用して保留になっている注文を調査し、それを承認または否認できます。

Storefront の使用

Storefront の表示

ブラウザを使用して、次の URL を表示してください。

http://localhost:8000/petstore


新規ユーザの作成

初めてお店を訪れた場合には、商品を購入する前に新規ユーザを作成します。

  1. ホームページの「ログイン」をクリックします。

  2. 「新規にユーザ登録する場合はこちらに必要事項を入力してください。」というフォームに必要事項を入力して、「新規ユーザ登録」をクリックします。

  3. アカウント情報を入力して、「送信」をクリックします。

注文する

たとえば、ブルドッグを購入するには、次の手順に従います。

  1. ホームページで、「犬」を選択します。

  2. 「ブルドッグ」をクリックします。

  3. 「雄 おとな ブルドッグ」の行の「カートに追加」をクリックします。

  4. 「チェックアウトへ」をクリックします。

    請求と出荷に関する情報のフォームが表示されます。ユーザの便宜のために、すでに情報が入力されています。

  5. 「送信」をクリックします。

ユーザが注文を完了すると、その注文は Storefront によって非同期的に OPC に送信されます。注文金額が $500 未満であれば、OPC はその注文を自動的に承認します。

管理クライアントの使用

管理クライアントを開く

注文を調べたり修正したりするには、Java 管理クライアントを使用します。

ブラウザを使用して、次の URL を表示してください。

http://localhost:8000/admin/


管理者のリッチクライアントインタフェースを起動するには、ホーム画面の「Launch Rich Client」をクリックします。クライアントは、Java Web Start を使って起動されます。

クライアントが表示されると、保留中の注文とそれ以外の注文を表示するための 2 つのパネルがあります。先ほどのブルドッグの注文は、金額が $500 未満なので、承認済みの注文になります。$500 未満の注文はすべて自動的に承認されます。

保留以外の注文の表示

「View Non-Pending Orders」パネルには、保留中以外の注文が表示されます。このパネルに表示される注文は、承認、否認、または確定されたものです。注文を読み込むには、「Refresh Data」ボタンをクリックします。

Supplier に対して在庫を作成または編集するには、「Supplier の管理」を参照してください。

注文の承認/否認

保留中の注文を承認または否認するには、「Process Pending Orders」パネルを使用します。注文が保留された状態で表示されるのは、合計金額が $500 を超える場合です。

注文を承認または否認するには、次の手順を行います。

  1. 「Process Pending Orders」タブをクリックします。

  2. 該当する注文の行をクリックします。

  3. その行の「Status」セルをクリックします。

  4. 新しいステータスを選択します。

任意の数の注文について承認または否認を実行できます。変更を保存するには、「Commit」をクリックします。変更を確認するには、「Refresh Data」ボタンをクリックします。

Supplier の管理

Supplier クライアントを開く

Supplier とその在庫は、簡単な Web インタフェースを使って管理します。Supplier を使用するには、その前に必ず管理クライアントからログアウトしてください。

ブラウザを使用して、次の URL を表示してください。

http://localhost:8000/supplier/


在庫の表示と編集

在庫を表示および編集するには、ホーム画面の「Display Inventory」をクリックします。

各品目の在庫の数量が表形式で表示されます。品目の数量を編集するには、その品目エントリの「New Quantity」セルを編集して、「Update」ボックスをチェックします。すべての変更を入力した後、「Submit」をクリックして、編集内容をデータベースにコミットします。

データベースが更新されるたびに、Supplier は在庫の変更を OPC に通知します。OPC は、その変更内容を確認し、以前には充当できなかった注文に対して商品を充当できるかどうかを調べます。もし注文に商品を充当できる場合は、その注文を完了済みとしてマークします。
