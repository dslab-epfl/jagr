デモの Web サービスの有効化

ここでは、Java Pet Store Demo の Web サービス機能を有効にする方法について説明します。デモの Web サービス機能を有効にするには、Java 2 SDK, Enterprise Edition (J2EETM SDK), v1.3.1 が必要です。Web サービスを Java 2 SDK, Enterprise Edition (J2EETM SDK), v1.4 に配備する場合は、BluePrints の Web サイトを参照してください。また、「デモのインストール」の説明に従って、まずデモの基本バージョンをインストールしておく必要もあります。

システム要件

次の付加的なソフトウェアをダウンロードしてインストールする必要があります。

    1. Java Web Services Developer Pack, v1.0_01: ダウンロード

必要な環境変数

次の付加的な環境変数を設定する必要があります。

    JWSDP_HOME 変数に、Java Web Services Developer Pack (JWSDP) のインストール先のルートディレクトリへのパスを指定する必要があります。たとえば、JWSDP を /usr/local/java/jwsdp-1_0_01 にインストールした場合は、そのパスを JWSDP_HOME の値として設定する必要があります。このドキュメントの以降の部分では、このパスを <jwsdp.home> と表記します。

デモのインストール

Web サービスを有効にしてデモをインストールするには、次の手順に従ってください。

Web サービステクノロジを J2EE に統合するスクリプトを実行します。

UNIX を使用している場合は、シェルプロンプトを開いて次のコマンドを入力します。

cd <jwsdp.home>/bin
sh jwsdponj2ee.sh $J2EE_HOME

Windows を使用している場合は、コマンドプロンプトを開き、次のコマンドを入力します。

cd <jwsdp.home>\bin
jwsdponj2ee.bat %J2EE_HOME%

Cloudscape データベースを起動します。

UNIX を使用している場合は、シェルプロンプトを開き、次のコマンドを入力します。

cd $J2EE_HOME/bin
cloudscape -start
Windows を使用している場合は、コマンドプロンプトを開き、次のコマンドを入力します。 
cd %J2EE_HOME%\bin
cloudscape.bat -start
J2EE サーバを起動します。

UNIX を使用している場合は、シェルプロンプトを開き、次のコマンドを入力します。

cd $J2EE_HOME/bin
j2ee -verbose
Windows を使用している場合は、コマンドプロンプトを開き、次のコマンドを入力します。 
cd %J2EE_HOME%\bin
j2ee.bat -verbose
    Web サービスを有効にした Pet Store のディレクトリに移動します。

    UNIX を使用している場合は、シェルプロンプトを開き、次のコマンドを入力します。

    cd <petstore.home>/webservices
    Windows を使用している場合は、コマンドプロンプトを開き、次のコマンドを入力します。 
    cd <petstore.home>\webservices
    設定スクリプトを実行して、アプリケーションを配備します。

    UNIX を使用している場合は、シェルプロンプトを開き、次のコマンドを入力します。

    sh setup.sh deploy
    Windows を使用している場合は、コマンドプロンプトを開き、次のコマンドを入力します。 
    setup.bat deploy
デモをインストールした後は、Pet Store の使用を開始できます。

Web サービスを有効にした Java Pet Store Demo アプリケーションに加えられた変更についての詳細は、このページを参照してください。
